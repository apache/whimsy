This tool helps file documents received via fax or email.


  1. Documents to be filed appear in the Work List area.

    1. Documents that are received via fax appear as .pdf files with an eFax prefix. These documents can be processed directly. There is no metadata associated with these documents, so all information must be entered manually.

    2. Documents that are received via email are in one of two forms. Metadata associated with the documents includes the sender's email address and name. This metadata will populate certain forms for specific document types.

      1. If an email is received with a single document to be processed that document will appear by itself.

      2. If an email is received with multiple documents, all of the documents will be put into a single directory and the directory appears in the work list.

  2. To convert documents to a form that can be processed, select the directory from the Work List. The contents of the directory will be displayed on the view port.

    1. If the documents are a file and a signature (e.g. icla.txt and icla.txt.asc) the tool will attempt to verify the document via the command "gpg --verify icla.txt.asc". The results of the verification are displayed in the view port along with the list of documents. If the document verifies correctly, it can be processed as if it were a document.

      1. If the document does not verify because the public key is not available, try to download the key from a public key server via the command "gpg --recv-keys <pub-key>". After the key is downloaded, you can try again.

      2. If the document does not verify because the signature is BAD, contact the sender and attempt to get the document in a different mode.

    2. If the documents are a collection of documents, they can be turned into individual documents via the Staple command in the view port.

      1. If the documents comprise a single document, e.g. icla1.pdf, icla2.pdf, select both documents and press the Staple key. Both documents will be combined into a single .pdf document and displayed in the Work List. This technique currently works for .pdf and .jpg files.

      2. If the documents comprise multiple independent documents, select a single document and press Staple. The single document will be copied into a new document with the same metadata and placed into the Work List. This technique currently works for .pdf and .jpg files.

      3. For each document in the collection, after stapling the document is ready for processing.

      4. After all documents have been processed, check to make sure the directory has been deleted by using the "svn status" command. [The directory will not appear in the Work List if it is empty.] If not, remove it manually using the "svn rm" command.

  3. To process a document, select it from the Work List. If it is displayable, the document will appear in the view port. All documents must be complete and legible. Multipage documents must have all pages included (specifically, not just the first and last page) in the same transmission.

    1. ICLAs: Required fields are full name, email address, signature, and date. If there is already an existing ICLA for the same name, you must establish whether the existing document is for the same person.

      1. If the existing document is for the same person, process the new ICLA with a different file name (e.g. existing-name2.pdf) and before completing the process, create a directory (e.g. existing-name/) and move both existing-name.pdf and existing-name2.pdf into the new directory.

      2. If the existing document is for a different person, contact the submitter and see if there is an additional (middle) name that can be used in the "Full Name" section. If not, see if there is a title that can be used to distinguish the entries.

      3. Try to determine if the ICLA is from a person who should be given commit access. Read the original email containing the document, look at the "notify PMC" field on the form, and search the email records to determine if the ICLA is in response to an invitation from a PMC or PPMC. If there is a valid invitation from a PMC, select the url of the invitation so it can be pasted into the ICLA form below the "file" button.

    2. CCLAs must be signed by an officer of the corporation authorized to enter into binding contracts. Required fields are corporation name, contact, and email address.

    3. NDAs require the apache id.

  4. If there are duplicate documents to be processed, after processing the best document, remove the duplicates before committing. Choose "other" "junk" from the menu. After removing all duplicates, commit the batch.

  5. After processing an ICLA, if the sender should have an account created, select "New Account" from the menu. The last ICLA should appear in the "ASF New Account Request" form. In "Comments" indicate "original committer" if the ICLA is from a new podling. Otherwise, leave it blank.

  6. Forms received by other than the above process should be added to the documents/received directory and added via "svn add". The document will then appear in the work list to be processed as above. Documents that are not in the proper format should be converted to .pdf before adding them.
